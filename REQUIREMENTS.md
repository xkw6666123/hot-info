# 热点信息差 项目需求文档

> 所有修改必须对照此文档，确保不遗漏任何功能。

---

## 1. 数据源要求

### 1.1 平台覆盖（13个，缺一不可）
- 百度热搜、知乎、B站、今日头条、澎湃新闻
- 华尔街见闻、财联社、凤凰网、贴吧、微博
- **抖音热搜**（全量，49-66条，不能只取10条）
- 公众号热点、博主追踪

### 1.2 抖音热搜
- 全量抓取（`limit` 不限），有多少条发多少条
- API: `douyin.com/aweme/v1/web/hot/search/list/`
- 本地模式必须启用（不能跳过）

### 1.3 博主视频（5人 × 3条 = 15条）
| 博主 | 平台 | 状态 |
|------|------|------|
| 网吧信息差 | 抖音 | ✅ F2抓取 |
| 阿七大型纪录片 | 抖音 | ✅ F2抓取 |
| 陈先生 | 抖音 | ✅ F2抓取 |
| 人类观察菌 | 抖音 | ✅ F2抓取 |
| 沙漠一之雕 | B站 | ✅ B站API + ASR |

- 每人必须恰好3条，不能多不能少
- 使用 F2 库 + Chrome Cookie 抓取（本地优���）
- 降级链: F2 → TikHub API → Playwright(PW)

## 2. 拆解页面功能

### 2.1 视频文案（content_intro）
- **必须有完整文案**
- 抖音博���：ASR语音转文字提取 → 存为 content_intro
- B站博主：MiMo API ASR提取 → 存为 content_intro
- 文案保护：任何情况下不得被覆盖为空或短于50字
- 示例（沙漠一之雕）：664字完整脚本，可展开查看

### 2.2 展示要求
- 展开阈值：80字（不是200字）
- 超过阈值显示"展开全文(X字)"按钮
- 点击展开原地展开，不跳转
- 点击标题跳转到原视频（抖音/B站）
- 右上角"🔗 查看原视频"链接

### 2.3 互动数据
- 点赞数、评论数必须真实（从F2 API获取）
- 不能全为0或固定值

## 3. 精选首页（社会热榜）

### 3.1 排序规则
- 按 socialScore() 综合评分排序
- 评分维度：情绪冲击词 + 争议性 + 平台社会权重 + 素人内容 + 互动量
- 华尔街见闻等财经源降权（权重=1）
- 微博/贴吧/百度热搜等社会源提权（权重=14-15）

### 3.2 板块结构
```
🔥 社会热榜（Top 15，按社交热度）
🎬 博主最新（15条，5博主×3）
📰 更多热点（其余内容）
```

---

## 4. 灵感库

### 4.1 基本要求
- 50条灵感，从热点中按 douyin_score 筛选
- 来源多样性：每源最多8条
- 5种博主风格，每种可独立复制

### 4.2 风格要求
- **网吧信息差**：极短标题(5-15字)+固定标签，不写长文
  - 标签：#青年创作者成长计划 #内容过于真实 #大学生 #热点
  - 真实风格参考：标题即内容，无正文，无emoji
- **阿七纪录片**：日期锚点型，"X月X日社会热点信息差"
- **陈先生**：大型纪录片风格，"大型纪录片之《XX》"
- **人类观察菌**：热点快报风格，带数据和时间线
- **沙漠一之雕**：快报连播风格，详细叙述新闻

### 4.3 可复制
- navigator.clipboard API，一键复制
- 降级方案：prompt + execCommand

## 5. 数据完整性要求

### 5.1 content_intro 保护
- 写入 data.json 前强制执行保护层
- 所有 blogger 条目 content_intro >= 50字
- 已有 ASR 文案永不被覆盖

### 5.2 数据持久化
- 旧数据救援：3天窗口
- 过期清理：新闻3天，博主3天
- ASR 迁移：按 aweme_id 匹配（不用 URL）

### 5.3 并发安全
- da.json 写入使用 os.replace() 原子操作
- GitHub Actions 设置 concurrency 避免并发
