# 热点信息差网站 - 商业级产品要求

## 项目概述
- **网站**: rediancha.online
- **GitHub**: xkw6666123/hot-info
- **部署**: Vercel 自动部署
- **更新频率**: 每3小时自动更新

## 核心功能要求

### 1. 数据采集
- ✅ **13+平台覆盖**: 微博、百度、抖音、B站、知乎、贴吧、头条、澎湃、华尔街见闻、财联社、凤凰、公众号
- ✅ **抖音热搜**: 全量输出，不限条数
- ✅ **博主视频**: 5位博主，每人3条最新视频
- ✅ **排序**: 按社交热度排序

### 2. 博主视频管理
- **博主列表**:
  - 网吧信息差
  - 阿七大型纪录片
  - 陈先生
  - 人类观察菌
  - 沙漠一之雕
- **视频数量**: 每人恰好3条
- **更新策略**: 新视频替换最旧视频
- **数据保留**: 保留15条历史数据作为救援机制

### 3. 文案提取（ASR）
- **质量要求**: 1:1完整转录，不截断
- **语言**: 简体中文（自动繁体转简体）
- **推荐方案**: Whisper base（质量好、速度快）
- **备用方案**: mimo-v2.5-asr（小米ASR）
- **阈值**: 文案<200字时自动触发ASR提取

### 4. 灵感库
- **风格学习**: 基于真实博主文案风格
- **模板类型**: 4-5种（80-150字）
- **生成频率**: 每次更新时自动生成
- **质量要求**: 模仿真实人类叙述，无广告感

### 5. 数据展示
- **视频信息**: 标题、文案、真实互动数据（点赞、评论、分享）
- **发布时间**: 精确到天
- **排序**: 按社交热度优先

## 技术架构

### 前端
- **框架**: 纯HTML/CSS/JS
- **数据嵌入**: data.js 内联到 index.html
- **部署**: Vercel 自动部署

### 后端
- **数据采集**: Python脚本（generate_hot.py）
- **ASR提取**: Whisper base + mimo-v2.5-asr
- **数据处理**: OpenCC繁体转简体
- **定时任务**: GitHub Actions（每3小时）

### 数据存储
- **主数据**: data.json
- **JS数据**: data.js（嵌入HTML）
- **学习数据**: blogger_content_learn.json

## 工作流程

### 自动更新流程（每3小时）
1. **数据采集**: 抓取13+平台热点数据
2. **博主视频**: F2获取最新视频（sort_type=0按时间排序）
3. **ASR提取**: Whisper base提取文案
4. **数据处理**: OpenCC繁体转简体
5. **灵感生成**: 基于学习结果生成灵感库
6. **完整性检查**: 验证数据完整性
7. **部署**: 推送到GitHub，Vercel自动部署

### 手动干预流程
1. **检查问题**: 查看GitHub Actions日志
2. **本地修复**: 运行extract_content.py
3. **测试验证**: 检查文案质量
4. **推送部署**: git push触发自动部署
